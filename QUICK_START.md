# 快速开始指南

## 新增功能速览

### ✅ 1. API 供应商兼容性配置

**问题**: 之前只能通过 URL 模式（如 `/anthropic`）自动检测 API 类型，不够灵活。

**解决方案**: 现在可以显式指定 API 兼容类型。

**三种选项**:
- 🔄 **Auto (自动)**: 自动检测（默认，向后兼容）
- 🔵 **OpenAI 兼容**: 使用 OpenAI API 格式
- 🟣 **Anthropic 兼容**: 使用 Anthropic API 格式

**配置位置**: 设置 → 网络配置 → 其他AI服务 → API兼容类型

---

### ✅ 2. 增强的多模态文件存储

**问题**: 之前只支持图片，缺少元数据管理。

**解决方案**: 现在支持多种文件类型和完整的元数据。

**支持的文件类型**:
- 🖼️ **图片**: jpg, png, gif, bmp, svg, webp, avif
- 📄 **文档**: pdf, doc, docx, txt, md, csv
- 🎵 **音频**: mp3, wav, ogg, flac, aac
- 🎬 **视频**: mp4, avi, mov, wmv, webm

**新增功能**:
- 文件元数据（名称、类型、大小、MIME 等）
- 智能文件类型检测
- 标签和描述
- 按类型查询
- 元数据更新

## 5 分钟快速上手

### 步骤 1: 配置 Anthropic Claude API

1. 打开应用，进入设置
2. 点击"网络配置"
3. 在"其他AI服务"区域点击"增加"
4. 填写信息：
   ```
   名称: Claude API
   接口地址: https://api.anthropic.com/v1
   API兼容类型: Anthropic 兼容 ⭐ 新选项
   ```
5. 保存后，在"秘钥配置"中添加您的 Anthropic API Key
6. 选择该服务作为当前 AI 提供者
7. 开始对话！

### 步骤 2: 上传和管理多模态文件

```typescript
// 在任何组件中使用
import { ImageStore } from '@/core/db/ImageDb';

const store = ImageStore.getInstance();

// 保存文件（支持任何类型）
const fileId = store.saveMultimodalFile(file, {
  fileName: 'my-document.pdf',
  description: 'Important document',
  tags: ['work', 'important']
});

// 获取文件
const file = await store.getMultimodalFile(fileId);
console.log(file.metadata.fileName); // 'my-document.pdf'
console.log(file.metadata.fileType); // 'document'

// 查询所有图片
const images = await store.queryFilesByType('image');
```

## 常见用例

### 🎯 用例 1: 同时使用 OpenAI 和 Anthropic

```typescript
// 配置两个服务
const servers = [
  {
    key: 'openai-service',
    name: 'OpenAI GPT-4',
    url: 'https://api.openai.com/v1',
    vendorType: 'openai',  // 明确指定
    hasToken: true
  },
  {
    key: 'anthropic-service',
    name: 'Claude 3',
    url: 'https://api.anthropic.com/v1',
    vendorType: 'anthropic',  // 明确指定
    hasToken: true
  }
];

KeyValueData.instance().setaiServerList(servers);
```

很多服务提供 OpenAI 兼容的 API，例如：

### 🎯 用例 3: 在对话中发送图片

```typescript

```

## API 速查表

### ImageStore 方法

| 方法 | 说明 | 示例 |
|------|------|------|
| `saveMultimodalFile(data, options)` | 保存文件 | `store.saveMultimodalFile(blob, {fileName: 'test.pdf'})` |
| `saveImage(data)` | 保存图片（兼容旧API） | `store.saveImage(base64)` |
| `getMultimodalFile(id)` | 获取完整文件对象 | `await store.getMultimodalFile(id)` |
| `getImage(id)` | 获取文件数据（兼容旧API） | `await store.getImage(id)` |
| `queryFilesByType(type)` | 按类型查询 | `await store.queryFilesByType('image')` |
| `updateMetadata(id, updates)` | 更新元数据 | `await store.updateMetadata(id, {tags: ['new']})` |
| `deleteImage(ids)` | 删除文件 | `store.deleteImage([id1, id2])` |

### 文件类型

## 故障排除

### ❓ 问题: 配置了 Anthropic 但仍然报错

**检查清单**:
1. ✅ 确认 `vendorType` 设置为 `'anthropic'`
2. ✅ 确认 API Key 正确（Anthropic 使用 `x-api-key`）
3. ✅ 确认 URL 正确（应该是 `https://api.anthropic.com/v1`）
4. ✅ 检查浏览器控制台是否有 CORS 错误

### ❓ 问题: 文件保存后找不到

**解决方案**:
```typescript
// 确保使用正确的 ID
const fileId = store.saveMultimodalFile(file);
console.log('Saved with ID:', fileId); // 记录 ID

// 使用相同的 ID 检索
const file = await store.getMultimodalFile(fileId);
```

### ❓ 问题: 旧代码是否仍然有效？

**答案**: ✅ 是的！完全向后兼容。

```typescript
// 这些旧 API 仍然可用
const id = store.saveImage(data);
const data = await store.getImage(id);

// 但可以升级到新 API 以获得更多功能
const id = store.saveMultimodalFile(data, options);
const file = await store.getMultimodalFile(id);
```
